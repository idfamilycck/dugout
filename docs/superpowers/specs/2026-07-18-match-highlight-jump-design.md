# 경기 화면 하이라이트 점프 개편 — 설계 (2026-07-18, 사용자 승인)

## 배경 (사용자 피드백)
1. 라이브 피치에 선수 이름이 없다.
2. 선수들이 움직이지 않는다.
3. 경기를 전부 틀어주는 것이 아니라 "전술이 승리를 만드는 과정"을 핵심 장면 중심으로 보여줘야 한다.

사용자 선택: 하이라이트 점프 구조 / 22명 전원 이름 표시 / 장면 연출 중심 움직임 / 개입은 자유 일시정지 유지.

## 접근 (승인됨: A안)
엔진·store·카운터팩추얼 불변식 무변경. match 페이지의 재생 스케줄러와 LivePitch만 개편.
(B: 사전 시뮬+스크립트, C: 엔진 장면 메타 — 리스크 대비 이득 없어 기각)

## 설계

### 1. 하이라이트 점프 재생
- 스킵 구간: 분당 ~110ms 초고속 틱. 장면 없는 분은 그냥 흘러간다.
- 장면 분(정지 대상): 해당 분에 슛 전개(`shot/goal/save/corner`)·`crisis`·`card`가 있으면 자동 "장면 모드" — 골 3.2초 / 그 외 1.8초 정지 후 자동 재개. 슛으로 이어지지 않은 단순 `chance` 분은 정지하지 않고 스킵 유지(중계 피드에만 기록) — 경기당 찬스 분이 ~28개라 전부 멈추면 "핵심 장면만"이라는 목적이 무너진다.
- 90분 체감 약 40초~1분대. 1x/2x/4x 버튼 제거, ▶/⏸ + 작전 변경만 유지 (개입 = 자유 일시정지, 현행 유지).
- 하프타임 자동 정지 모달, 위기 배너(개입 유도), 종료 오버레이 유지.
- 새로고침/재진입 시 이미 지나간 장면 재연출 금지 (마운트 시 최신 장면 분으로 커서 시드 — 기존 one-shot 패턴 준수).
- 작전 변경 시트 열림 → 진행/장면 타이머 정지, 활성 장면 해제.

### 2. 장면 자막 + 전술 귀속 (SceneOverlay)
- 헤드라인: 주 이벤트(우선순위 goal>save>shot>corner>chance>crisis>card)의 textKo + 분.
- 체인 표시: 같은 분의 장면 이벤트 나열 (찬스 → 슛 → 골).
- 전술 코멘트: 공격 장면(chance/shot/goal/corner)에 한해, 공격 측 기준 `applyModifiers()` 발동 규칙 중 deltaAttack 최대(>0)인 규칙의 textKo를 "발동 전술" 칩으로 인용. 엔진이 실제 λ에 반영한 규칙만 인용(정직한 귀속). 해당 규칙 없으면 칩 생략. crisis/card는 칩 없음.
- 개입(교체·전술 변경) 후에는 변경된 setup 기준으로 재계산.

### 3. 선수 이름 22명 전원 (LivePitch)
- 각 점 아래 이름 라벨(fontSize ~3.6), 등번호는 점 안 유지.
- 장면의 주 이벤트 playerId 점은 확대+펄스 강조.

### 4. 장면 연출 중심 움직임 (LivePitch)
- 평상시: 최근 장면성 이벤트 5개의 공수 균형으로 lean(-1..1) 계산 → 양 팀 진형 전체가 공 방향으로 ±6px 스프링 이동.
- 장면 중: 기존 3점 돌파+공 애니메이션 유지, 주인공 점 펄스.

### 5. 부수 변경
- e2e/smoke.spec.ts: 4배속 버튼 의존 제거, 자동 페이싱 기준으로 대기/타임아웃 조정.
- 순수 로직(`장면 감지/우선순위/체인/귀속/lean`)은 `components/match/scene.ts`로 분리해 vitest 단위 테스트.

## 테스트
- scene.ts 단위 테스트: 장면 감지(타입 필터), 주 이벤트 우선순위, 체인 순서, 귀속 규칙 선택(양수 deltaAttack 최대, 없으면 null), lean 클램프·방향.
- 브라우저 실측: 킥오프→스킵→장면 정지→자막/칩→재개→하프타임→종료, 새로고침 재진입 시 장면 재연출 없음, 콘솔 0.
- 기존 98개 유닛 + E2E 스모크 그린 유지.
