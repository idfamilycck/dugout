# "2026 월드컵 다시 쓰기" 모드 — 설계 (2026-07-20, 사용자 승인)

- 대상: TOUCHLINE(터치라인)에 실제 2026 월드컵 데이터 기반 리플레이·개입 모드 추가
- 상위 문서: `2026-07-17-touchline-tactics-simulator-design.md` (기존 자유 매치업 모드 — 유지)
- 마감 제약: 기획서 PDF 갱신 7/27(월) 10:00, 저장소 커밋 8/3(월) 10:00 이후 금지

## 1. 목적

실제 2026 월드컵 경기(총 104경기)의 타임라인을 재생하다가, 실점·퇴장 같은 **결정적 순간의 5분 전으로 돌아가** 사용자가 감독으로 개입하고, 그 분기의 시뮬레이션 결과를 **실제 역사와 나란히 비교·리뷰**하는 모드. 기존 가상 매치업 모드와 병행(홈에서 모드 분기).

## 2. 사용자 결정 사항 (확정)

| 결정 | 선택 |
|---|---|
| 경기 범위 | 전 104경기 (데이터 불량 경기는 노출 제외 허용) |
| 개입 방식 | 결정적 순간 카드 (자유 되감기 비범위) |
| 기존 모드 | 병행 — 새 모드 추가, 기존 기능 무변경 |
| 데이터 소스 | ESPN 공개 JSON 1회 수집 → 정적 JSON 커밋, 위키피디아로 스코어 대조 검증 |
| 선수 능력치 | 기존 원칙 유지 — 실명·출전 기록만 사실, 능력치는 가상 생성(고지 유지) |

## 3. 데이터 파이프라인 (`scripts/ingest-wc2026.mjs`, 1회 실행 후 산출물 커밋)

### 3.1 산출물 (`data/wc2026/`)
- **matches.json**: 104경기. 경기별 — id, 라운드(조별 A~L / 32강~결승), 팀 코드 쌍, 최종 스코어(연장·승부차기 포함), 경기장, 킥오프 일시, **이벤트 타임라인**(분 단위: 골/자책골/PK골, 교체(in/out), 옐로카드, 레드카드), **선발 11 + 교체 명단**(실명, 포지션)
- **teams.json**: 48개국 — 코드, 국문 명칭, 조 편성, 최종 성적(진출 라운드), 파생 ELO
- 결승(스페인 v 아르헨티나, 한국시간 7/20 오전 종료)은 데이터 확정 확인 후 수집 — 미확정 상태로는 커밋하지 않음

### 3.2 파생 데이터 규칙 (결정론적 — 스크립트에 하드코딩, 재실행 시 동일 출력)
- **팀 ELO**: 진출 라운드 기반 계단식 산출(우승 최상 ~ 조별 탈락 최하) + 조별 승점 미세 보정. 기존 16개국 자유 모드 ELO와 별도 필드(모드 간 간섭 없음)
- **선수 가상 능력치**: 팀 ELO 스케일 × 포지션 프로파일(기존 `roles` 가중 체계 재사용) × 이름 해시 기반 소폭 분산 × 대회 출전 시간 가중(주전 보정). 기존 8종 능력치 스키마와 동일 형태로 생성 → 엔진 무수정 소비
- 실명 사용 범위: 이름·포지션·출전/교체/득점/카드 기록(공표된 사실 데이터)만. 사진·초상 미사용, "모든 능력치는 가상" 고지를 실경기 모드 화면에도 노출

### 3.3 정합성 게이트 (vitest, `data/wc2026` 대상)
- 스키마 밸리데이션(필수 필드·타입·팀 코드 참조 무결성)
- 경기별 이벤트 골 수 합계 = 최종 스코어 (연장 포함, 승부차기 제외)
- 교체 ≤ 팀당 5회(연장 6회째 허용), 레드카드 후 해당 선수 이벤트 부재
- 선발 11인 정확히 11명, 교체 in 선수는 명단 내 존재
- 게이트 실패 경기는 `excluded: true` 마킹 → UI 미노출 (전 경기 노출을 강제하지 않음)

## 4. 사용 흐름 & 화면

### 4.1 홈 — 모드 분기
- 기존 퀵스타트·자유 매치업 위에 모드 스위치: **"자유 매치업"** / **"2026 월드컵 다시 쓰기"**
- 다시 쓰기 진입 → 경기 브라우저: 라운드 필터(조별/32강/16강/8강/4강/결승), 한국 경기 최상단 고정, 경기 카드에 실제 스코어·라운드·"결정적 순간 N개" 배지

### 4.2 경기 상세 — 결정적 순간 선택 (신규 라우트 `/rewrite`)
- 지휘할 팀 선택(양 팀 모두 가능) → 실제 타임라인 요약(골·카드·교체 마커)
- **결정적 순간 자동 추출 규칙** (지휘 팀 관점, 정규 90분 내로 한정):
  1. 실점(상대 골) — 항상
  2. 지휘 팀 퇴장(레드카드)
  3. 리드 상실 시점(동점·역전 허용골) — 1과 중복 시 하나의 카드로 합침(라벨만 강화)
  4. 75분 이후 동점 상황 진입(승부처)
- 카드 문구 예: "88' 결승골 실점 — 5분 전(83')으로 돌아가시겠습니까?"
- 개입 시점 = max(이벤트 분 − 5, 0). 연장전(90'+) 순간은 추출 대상에서 제외(비범위)

### 4.3 작전실 축약판 — 시점 상태 복원
- 선택 시점의 **실제 상태 주입**: 스코어, 해당 분까지의 교체가 반영된 현재 11인(퇴장 시 10인 그대로), 카드 상황, 남은 교체 슬롯(실제 사용분 차감)
- 기존 전술 3계층 UI 재사용(포메이션·지시·역할·특수 지시) + 교체 지시
- 11인 미만 시작 차단 검증은 실경기 모드에서 "실제 상태 그대로" 예외 허용(퇴장 10인 시뮬 가능)

### 4.4 경기 — 분기 시뮬레이션
- 기존 하이라이트 점프 재생 그대로, 개입 시점부터 잔여 시간(~90')만 시뮬
- 토너먼트 경기에서 90' 종료 동점이면 기존 승부차기 미니게임 제안(연장전 시뮬은 비범위)

### 4.5 복기 — 실제 역사 vs 나의 평행세계
- 상단 비교 카드: "실제: 1-2 패배" vs "당신의 지휘: 2-2 → 승부차기 승리" 형태
- 타임라인 병렬 표시: 개입 시점 이전(공유 구간)은 실제 이벤트, 이후는 실제/시뮬 두 줄
- 기존 전술 평가·근거 카드 패널 재사용. 카운터팩추얼 문구는 실경기 모드에서 "실제 역사 대비 델타"로 표기

## 5. 엔진·상태 확장 (기존 재사용 극대화)

- **`lib/engine/rewrite.ts` (신규, 순수 TS)**: 실경기 상태 → 엔진 시뮬 초기 상태 어댑터 `fromRealState(match, side, minute)` — 스코어·라인업·체력(경과 분 기반 감쇠 적용)·남은 교체 수 주입. 시뮬 자체는 기존 `match.ts` 재사용(시작 분 파라미터화)
- **store**: `mode: 'free' | 'rewrite'` + `rewriteContext { matchId, side, momentId, takeoverMinute }`. persist 스키마 버전 업(`touchline-v2`), 기존 자유 모드 플로우는 무변경
- 라우트: `/rewrite`(경기·순간 선택) 신규, `/tactics` `/match` `/result`는 mode 분기로 재사용
